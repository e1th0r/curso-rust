# 01 - Fundamentals: Cargo and your first program

## What is cargo

cargo is Rust's project manager. It replaces several tools that in other
languages are separate (npm, pip, make, etc.). With a single command you
can create projects, compile, run, and manage dependencies.

## Basic commands

Create a new project:

    cargo new project-name

Compile the project:

    cargo build

Compile and run in one step:

    cargo run

Check for errors without fully compiling (faster):

    cargo check

Compile in production mode (optimized):

    cargo build --release

## Structure of a cargo project

When running cargo new hello-rust, the output was:

    Creating binary (application) `hello-rust` package

And it generated this structure:

    hello-rust/
    ├── Cargo.toml       <- project configuration
    ├── Cargo.lock       <- exact dependency versions (auto-generated)
    └── src/
        └── main.rs      <- main source file

## Cargo.toml

Actual content generated by cargo new:

    [package]
    name = "hello-rust"
    version = "0.1.0"
    edition = "2024"

    [dependencies]

- [package]: project metadata
- edition: Rust language version (2024 is the most recent)
- [dependencies]: external libraries the project needs (empty = none)

## Your first program

Actual content of src/main.rs generated by cargo new:

    fn main() {
        println!("Hello, world!");
    }

- fn main(): program entry point, the first thing that runs
- println!: macro that prints text to screen (the ! indicates it is a macro,
  not a regular function — the difference will be covered later)
- The curly braces { } delimit the function body

## Running the program

When running cargo run, the output was:

    Compiling hello-rust v0.1.0 (/home/hector/curso-rust/01-fundamentos/hello-rust)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.65s
    Running `target/debug/hello-rust`
    Hello, world!

- Compiling: compiled the source code
- Finished 'dev' profile [unoptimized + debuginfo]: development mode, faster
  to compile but without optimizations (for production: cargo build --release)
- Running: executed the generated binary
- Hello, world!: program output

## The generated executable

The binary is located at target/debug/:

    -rwxr-xr-x 2 hector hector 4345152 Jun 21 23:02 target/debug/hello-rust

4.3 MB in development mode (unoptimized + debuginfo). This size is expected
and should not be a concern: it includes debug symbols and applies no
optimizations, which helps with debugging but produces larger binaries.
When compiling for production the result is considerably smaller:

    cargo build --release
    (binary located at target/release/hello-rust)

The target/ folder is not uploaded to the repository (it is in .gitignore).

## Note on git and cargo

cargo new automatically initializes a git repository inside the project.
If the project lives inside another git repository (like this course), you
must delete that inner .git to avoid conflicts:

    rm -rf project-name/.git
